# Maintainers

This document defines who reviews what in this repository.

## Core Maintainers

| Role | GitHub |
|------|--------|
| Project Lead | [@MakAkasha](https://github.com/MakAkasha) |
| Hardware Lead | open position |
| Firmware Lead | open position |
| Software Lead | open position |
| Documentation Lead | open position |

> Domain leads are open positions. If you are an experienced engineer in one of these areas and want to take ownership of a subsystem, open a GitHub Discussion or reach out via [Discord](https://discord.gg/HRYwPTB9).

## Review Ownership

- `hardware/**` → Hardware Lead + Project Lead
- `firmware/**` → Firmware Lead + Project Lead
- `software/**` → Software Lead + Project Lead
- `docs/**` and `*.md` → Documentation Lead + Project Lead

## Review Rules

- At least 1 approval from a relevant domain maintainer
- Safety-critical changes require 2 approvals
- Major architecture changes should include an issue discussion link
