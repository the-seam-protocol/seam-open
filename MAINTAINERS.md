# Maintainers

This document defines who reviews what in this repository.

## Core Maintainers

> Replace placeholders with GitHub usernames.

- Project Lead: `@your-handle`
- Hardware Lead: `@hardware-lead`
- Firmware Lead: `@firmware-lead`
- Software Lead: `@software-lead`
- Documentation Lead: `@docs-lead`

## Review Ownership

- `hardware/**` → Hardware Lead + Project Lead
- `firmware/**` → Firmware Lead + Project Lead
- `software/**` → Software Lead + Project Lead
- `docs/**` and `*.md` → Documentation Lead + Project Lead

## Review Rules

- At least 1 approval from a relevant domain maintainer
- Safety-critical changes require 2 approvals
- Major architecture changes should include an issue discussion link
