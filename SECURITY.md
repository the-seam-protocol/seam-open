# Security Policy

## Reporting a Vulnerability

Please do **not** open public issues for security vulnerabilities.

Instead, report privately to the maintainers listed in [MAINTAINERS.md](./MAINTAINERS.md) with:
- A clear description of the vulnerability
- Affected components/paths
- Reproduction steps or proof of concept
- Suggested mitigation (if available)

We will acknowledge receipt as soon as possible and work with you on coordinated disclosure.

## Scope

This policy applies to all repository contents, including software, firmware, hardware design files, and documentation that could impact operational safety.

## Disclosure Process

1. Private report received
2. Maintainers validate and assess severity
3. Fix prepared and reviewed
4. Public disclosure with remediation guidance
