# Security Policy

## Reporting a Vulnerability

Please do **not** open public issues for security vulnerabilities.

Report privately using one of these channels:

- **Email**: security@seam-protocol.sd
- **GitHub Private Advisory**: [Report a vulnerability](https://github.com/the-seam-protocol/seam-open/security/advisories/new)

Include in your report:
- A clear description of the vulnerability
- Affected components or file paths
- Reproduction steps or proof of concept
- Suggested mitigation (if available)

We will acknowledge receipt within 48 hours and work with you on coordinated disclosure.

## Scope

This policy covers all repository contents — software, firmware, hardware designs, and documentation — where a flaw could impact operational safety or user security.

## Disclosure Process

1. Private report received
2. Maintainers validate and assess severity
3. Fix prepared, reviewed, and merged
4. Public disclosure issued with remediation guidance

## Out of Scope

- Theoretical vulnerabilities with no practical attack path
- Issues in third-party dependencies (report to the upstream project instead)
