# Support

## How to get help

- **GitHub Issues**: Bug reports and feature requests
- **GitHub Discussions**: Design questions, architecture discussion, and contributor coordination

## Before opening an issue

1. Search existing issues/discussions
2. Check `README.md`, `docs/getting-started.md`, and `CONTRIBUTING.md`
3. Include clear reproduction steps (for bugs)

## Security issues

Please follow [SECURITY.md](./SECURITY.md) and do not report vulnerabilities in public issues.
